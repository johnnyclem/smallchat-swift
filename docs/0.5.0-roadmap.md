# smallchat-swift 0.5.0 Roadmap

## Context

The TypeScript `@smallchat/core` reference implementation has moved well past
Swift's last release (0.3.0). As of `main` on 2026-04-30 it is at version
0.4.0 with a substantial unreleased trail of work on top — including a brand-new
external compile target, `muhnehh/loom-mcp`. This roadmap captures everything
that needs to ship in `smallchat-swift` 0.5.0 to bring the Swift port back to
feature parity with TypeScript main, and breaks the work into phases that can
each land independently without breaking the build.

## Source of truth

- TypeScript `package.json`: 0.4.0
- TypeScript main HEAD: `5ac6952` "Add loom-mcp manifest as a compile target (#61)"

## Phasing

### Phase 1 — loom-mcp manifest support (foundation)

**Goal:** the Swift compiler ingests the same `loom-mcp-manifest.json` the
TS compiler does, with `compilerHints`, `aliases`, `selectorHint`, and
`namespace` flowing all the way into the embedding text and dispatch table.

Changes:

- `Sources/SmallChatCore/Types/ProviderManifest.swift`
  - Add `description: String?`
  - Add `compilerHints: ProviderCompilerHints?` (type already exists in
    `SmallChatManifest.swift`)
- `Sources/SmallChatCore/Types/ToolDefinition.swift`
  - Add `compilerHints: CompilerHint?`
- `Sources/SmallChatCompiler/Parser.swift`
  - Propagate provider + tool hints into `ParsedTool`
  - When embedding, fold `selectorHint`, `aliases`, and provider
    `semanticContext` into the text we hand to the embedder so that natural-
    language phrases like "find callers of foo" hit `loom_find_importers`
- `Sources/SmallChatCompiler/ToolCompiler.swift`
  - Honor `priority`, `preferred`, `exclude` from per-tool hints
- `examples/loom-mcp-manifest.json` — port from TS
- `Tests/SmallChatCompilerTests/LoomManifestTests.swift` — round-trip + alias-
  driven resolution

### Phase 2 — confidence-tiered dispatch (TS PR #54, the big one)

Changes:

- New `Sources/SmallChatCore/Types/DispatchConfidence.swift` —
  `enum DispatchTier { case exact, high, medium, low, none }` with thresholds
- `Sources/SmallChatRuntime/Dispatch.swift` — return tiered outcomes,
  emit per-step proof traces, drop default vector threshold from 0.75 → 0.60
- `Sources/SmallChatRuntime/Verification.swift` (new) — pre-flight
  `respondsToSelector:` checks (schema validation, keyword overlap, optional
  LLM verification) for MEDIUM
- `Sources/SmallChatRuntime/Decomposition.swift` (new) — split LOW intents
  into ordered sub-intents with dependency resolution
- `Sources/SmallChatRuntime/Refinement.swift` (new) — NONE-tier handler
  returning structured `tool_refinement_needed` MCP results
- `Sources/SmallChatRuntime/LLMClient.swift` (new) — pluggable interface +
  graceful-degradation no-op default
- `Sources/SmallChatRuntime/DispatchObserver.swift` (new) — KVO-style
  observer that adapts thresholds from corrections
- `Sources/SmallChatMCP/MCPRouter.swift` — surface `tool_refinement_needed`
- `Sources/SmallChatCLI/Commands/CompileCommand.swift` — `--strict` flag
- `Sources/SmallChatCompiler/ToolCompiler.swift` — extend collision firewall
  to the full 0.75–0.95 zone (already partly there) and adapt-per-class
- Tests across `SmallChatRuntimeTests`, `SmallChatMCPTests`,
  `SmallChatCompilerTests`

### Phase 3 — Registry / Bundle / Install schemas (TS PR #52)

Changes:

- `Sources/SmallChatCore/Types/Registry.swift` (new): `RegistryEntry`,
  `SmallChatBundle`, `RegistryIndex`, `InstallPlan`
- `Sources/SmallChatCore/Schemas/` — JSON Schema files for validation
- `examples/registry/` — three example registry entries (GitHub, Slack,
  PostgreSQL) and an example bundle, mirroring TS PR #52
- `Sources/SmallChatCLI/Commands/InstallCommand.swift` (new) — render an
  `InstallPlan` for review before applying
- Tests: `RegistryEntryTests`, `BundleTests`, `InstallPlanTests`

### Phase 4 — new modules (TS PRs #55–#58, #60)

Each is a standalone Swift package target.

- **`SmallChatShorthand`** (PR #58) — primitives extracted from importance,
  CRDT, and compaction; ports the TS `@shorthand/core` package
- **`SmallChatImportance`** (PR #55) — three-signal importance detector
  (recency, centrality, semantic novelty)
- **`SmallChatCRDT`** (PR #56) — CRDT-based shared memory for multi-agent
  use, with conflict resolution
- **`SmallChatCompaction`** (PR #57) — three-strategy compaction
  verification (resampling, contradiction, diff invariants)
- **`SmallChatMemex`** (PR #60) — knowledge-base compiler with the same
  Read → Extract → Embed → Link → Emit pipeline as `ToolCompiler`. Adds
  `smallchat memex {compile,query,lint,inspect,export}` to the CLI

`Package.swift` gains five new library targets and matching test targets.

### Phase 5 — loom runtime client + GUI updates

Changes:

- `Sources/SmallChatTransport/Loom/LoomMCPClient.swift` (new) — thin wrapper
  around the existing `MCPStdioTransport` configured to spawn
  `npx @loom-mcp/server`. Surfaces a typed view of the 28 loom tools
- `Sources/SmallChatCLI/Commands/SetupCommand.swift` — auto-detect
  `@loom-mcp/server` on PATH and offer to register it
- `Sources/SmallChatApp/Views/`
  - `CompilerView.swift` — surface confidence tiers + collision warnings
  - `ResolverView.swift` — show proof traces and `tool_refinement_needed`
    suggestions
  - `DiscoveryView.swift` — list registry entries (Phase 3) and the
    pre-bundled loom manifest
  - `MemexView.swift` (new) — Memex compile/query inspection
- `Sources/SmallChatApp/AppState.swift` — wire new sections

## Verification

End-to-end sanity once Phase 1 lands:

```bash
swift build
swift test --filter SmallChatCompilerTests
swift run smallchat compile --source examples/loom-mcp-manifest.json --output /tmp/loom.toolkit.json
swift run smallchat resolve --toolkit /tmp/loom.toolkit.json "find callers of foo"
# expect: loom_find_importers, high confidence
```

Once Phase 5 lands, additionally:

```bash
swift run smallchat serve --toolkit /tmp/loom.toolkit.json &
# in another shell
curl -s http://localhost:3000/mcp/tools/list | jq '.tools | length'
# expect: 28
```
